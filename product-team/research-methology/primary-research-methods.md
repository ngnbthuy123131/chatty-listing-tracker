## 1. Customer Interviews

### Preparation Actions
- [ ] Define research questions (max 5 core questions per interview) - GG Meet
- [ ] Create interview script with JTBD framing
- [ ] Set up recording system (Loom with permission)
- [ ] Build interview tracking sheet (date, segment, key insights)

### Recruitment Actions
- [ ] Segment your existing users by behavior (power users, churned, new)
- [ ] Pull list of 50 merchants who installed competitor apps alongside yours
- [ ] Offer incentive ($20-50 gift card or account credit)
- [ ] Send personalized outreach (aim for 10-15 completed interviews)

### Interview Execution
- [ ] Ask: "What were you trying to accomplish when you installed Chatty?"
- [ ] Ask: "Walk me through the last time you [did the job]"
- [ ] Ask: "What's frustrating about your current solution?"
- [ ] Ask: "If you could wave a magic wand, what would change?"
- [ ] Probe on switching triggers: "What would make you switch tools?"

### Post-Interview Actions
- [ ] Transcribe within 24 hours (use Otter.ai or similar)
- [ ] Tag quotes by theme in a central repository
- [ ] Share key insights with product team weekly

## 2. Win/Loss Analysis

### Setup Actions
- [ ] Add "competitor mentioned" field to CRM/support system
- [ ] Create win/loss interview template (5 min call or survey)
- [ ] Train sales/support to flag competitive losses in real-time
- [ ] Set up automated survey trigger when deal is lost

### Data Collection Points
- [ ] Lost deals: "Which solution did you choose and why?"
- [ ] Churned users: "What are you using now instead?"
- [ ] Won deals: "What other options did you consider?"
- [ ] Trial abandons: "What stopped you from converting?"

### Questions to Ask (Lost Deals)
- [ ] "What was the #1 reason you chose [competitor]?"
- [ ] "Was there anything we could have done differently?"
- [ ] "How did you first hear about [competitor]?"
- [ ] "What features were decisive in your choice?"

### Analysis Actions
- [ ] Categorize losses by reason (price, features, support, brand)
- [ ] Track win rate against each competitor monthly
- [ ] Identify patterns: which segments do we lose most?
- [ ] Report findings to leadership quarterly

---

## 3. Sales Call Listening

### Infrastructure Setup
- [ ] Enable call recording in your sales tools (Gong, Chorus, or basic Zoom)
- [ ] Create tagging system for competitive mentions
- [ ] Build "competitor mention" Slack channel for real-time sharing
- [ ] Set up weekly review of flagged calls

### What to Listen For
- [ ] Direct competitor mentions ("We're also looking at Gorgias")
- [ ] Feature comparisons ("Does your tool do X like [competitor]?")
- [ ] Objections that imply competitor strength
- [ ] Pricing comparisons and anchoring

### Documentation Actions
- [ ] Log competitor mention with timestamp and context
- [ ] Capture exact customer language (don't paraphrase)
- [ ] Note the prospect's segment/size/use case
- [ ] Track which competitors appear in which deal stages

### Weekly Review Process
- [ ] Review 5-10 calls with competitive mentions
- [ ] Extract top 3 objections heard this week
- [ ] Update battlecards with new counter-arguments
- [ ] Share patterns with product team

---

## 4. Support Ticket Mining

### Setup Actions
- [ ] Create saved search/tag for competitor mentions in help desk
- [ ] Train CS team to tag tickets mentioning competitors
- [ ] Build monthly report on competitor-related tickets

### What to Extract
- [ ] Feature requests that reference competitor capabilities
- [ ] Complaints comparing your product unfavorably
- [ ] Questions about migration from/to competitors
- [ ] Integration requests that signal competitor ecosystem

### Analysis Cadence
- [ ] Weekly: Quick scan of new competitor mentions
- [ ] Monthly: Quantify mentions by competitor and theme
- [ ] Quarterly: Deep analysis of trends and product gaps
